What Is Efference?

Efference replaces hand-coded physics simulators with a large language model that reasons about the physical world. Given an image or scene description, it identifies objects, materials, and forces—then predicts what will happen next.

The result is a structured JSON output that downstream systems can use for physics-constrained image generation, robotics planning, or scene understanding.

Named after the neuroscience concept of efference copy—the brain's internal prediction of sensory consequences before an action occurs.

EfferenceOutput 
{
  "objects": [{
    "name": "red_ball",
    "material": "rubber",
    "mass_kg": 0.15,
    "state": "resting"
  }],
  "forces": [{
    "type": "gravity",
    "source": "earth",
    "target": "red_ball",
    "magnitude": 1.47
  }],
  "will_happen": [
    "The ball will remain at rest."
  ],
  "confidence": 0.95

How It Works

Four stages from pixels to physics.

1

Image Input

Upload an image or describe a scene in natural language.

2

Visual Encoding

DINO + CLIP extract spatial and semantic features from the scene.

3

Physics Brain

An LLM identifies objects, computes forces, and predicts trajectories.

4

Guided Generation

Physics constraints guide diffusion to produce physically-plausible outputs.

Features

Everything you need for physics-aware AI.

18

Material Types

Rubber, wood, metal, glass, ice, water, and 12 more—each with distinct physical properties.

10

Force Types

Gravity, normal, friction, tension, spring, drag, buoyancy, magnetic, applied, and centripetal.

Trajectory Prediction

Time-stamped 3D position sequences for every moving object in the scene.

Physics Constraints

Energy conservation, no-penetration, contact maintenance—8 constraint types with confidence scores.

Runs on Apple Silicon
